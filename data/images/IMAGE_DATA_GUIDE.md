# 圖片資料目錄說明

本目錄用於存放本專案處理流程中會用到的所有圖片檔案，分為三大區塊：

## 目錄結構

```
data/images/
├── reference/          # 參考檔案：作為分析基準、風格學習、評分依據的範例作品
│   ├── basic/          # 基本級參考作品
│   ├── intermediate/   # 中級參考作品
│   └── advanced/       # 高級參考作品
├── pending/            # 待處理：已匯入但尚未執行分析／修圖流程的圖片
└── completed/          # 完成：已完成分析與處理流程的圖片
```

## 分級說明

`reference/` 底下的三級（基本／中級／高級）**由使用者主觀判斷後手動分類放入**，不是由系統自動評分或演算法決定。這三級純粹是「參考作品的難度／等級分類」，用來當作分析基準與風格學習的樣本來源。

## 完成檔案的評分方式

`pending/` 內的圖片經系統分析／處理完畢後，移入 `completed/`；但**評分本身不透過資料夾層級表示**，而是寫入資料庫的評分系統（對應 Phase 5 的評分與風格學習機制，詳見 `docs/roadmap.md`）。也就是說：
- `reference/basic|intermediate|advanced`：使用者手動分級的參考樣本，資料夾即分級
- `completed/`：系統處理完成的檔案，其評分結果記錄在資料庫，不靠資料夾區分等級

## 流程慣例

1. 新素材先放入 `pending/`
2. 執行分析／處理流程後，搬移至 `completed/`，處理結果與評分寫入資料庫
3. 使用者判斷某張圖片適合作為分析基準或風格學習樣本時，手動歸類放入 `reference/` 對應等級

## 參數化路徑

上述所有路徑已抽離至 `config/settings.yaml` 的 `paths` 區塊，程式中應讀取設定檔取得路徑，不應在程式碼中寫死路徑字串（見專案 `CLAUDE.md` 第 3 節）。

## 注意事項

- 實際圖片檔案（`.jpg` / `.png` / `.raw` 等）不會被 git 追蹤（見專案 `.gitignore`），僅目錄結構本身（透過 `.gitkeep`）會被保留在版本控制中。
- 圖片實體檔案應另有備份機制（例如外接硬碟或 D: 磁碟區），不依賴 git 版本控制保存原始素材。
